# One Take Wonder Marketing - Codex Project Instructions

## Audience
Primary user is Palloma Earl (business owner, non-technical).

## Communication Style
- Use plain, friendly language.
- Do not require Git or terminal knowledge.
- Avoid technical jargon unless she asks for it.
- Keep instructions short and practical.

## Default Behavior
When user asks to "update" or "update the site", always do end-to-end delivery:
1. Make requested website changes.
2. Validate the result briefly.
3. Commit the changes.
4. Push to GitHub `main`.
5. Confirm in simple language what changed and that it is published.

## Deployment Context
- Site is static GitHub Pages site from repo root.
- Main files are `index.html`, `styles.css`, `Logo/Logo.png`, `MarketingBanner.jpg`, and `CNAME`.
- Custom domain is `onetakewondermarketing.com`.

## Safety Rules
- Never delete unrelated files.
- Never run destructive git commands unless explicitly requested.
- If credentials or permissions block push, clearly explain next step in plain language.
